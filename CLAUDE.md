# Outfittr

## Architecture
This is a MERN stack project with a Flutter mobile integration.

## Project Structure
- frontend folder for React frontend
- backend files are in root dealing with API calls and more
- mobile folder deals with everything Flutter related

## Rules
- Backend is source of truth
- Use existing API endpoints only
- Keep UI minimal until everything works
- No new architecture decisions without matching backend

## Style Guidelines
- When writing new code, add comments for functions or complicated lines explaining what it does
- Have open curly brackets ```{``` start below the function and not next to it
- Add a comment at the top of new files created that is the title of what it is or does

## Goal for now
The goal for now is to implement the flutter mobile integration since the frontend and backend is done for the most part.
